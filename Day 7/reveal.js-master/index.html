History Of CoverFlow

Conceived by: Andrew Coulter Enright.
Implented by: Indepedent Mac Developer Jonathan Del Strother in 2005.
Background: In 2006 Apple purchased the complete rights to CoverFlow. Apple then began to immediately use coverflow in their new OSX apps and itunes 7.0.

Timeline:

PHP + GD/ImageMagick (2006-2008)

The earliest attempts of implementing CoverFlow came greatly from server side image processing. The artwork would be stored on a server and read through PHP script. Which equals the image being processed and pushed back to the browser.

Timeline:

Flash (2007)

Disadvantages towards using PHP, it had limitations. Developers realzied that and turned to Adobe's Flash to implement the CF effect instead. Flash at the time was perfect for a decent framework use. However a non-plugin approach was more beneficial and desirable.

Timeline:

Jquery CSS3 CoverFlow (2008-2010)

Advantage for CF is that CSS supports multiple numbers of transformations. Using Jquery makes it possible to transform parameters required to make each cover art to the CF with a straight-forward effect allowing the user to skip to the next or previous cover with ease.

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
